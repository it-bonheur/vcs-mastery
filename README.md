# Developer Learning Lab: Version Control

This repository is a dedicated space for documenting my technical growth in **Version Control Systems (VCS)**, specifically focusing on **Git**. As a developer, I believe managing code history is just as important as writing the code itself.

## 🎯 Learning Objectives
- [ ] Master the **Git CLI** (beyond basic add/commit/push).
- [ ] Understand **Branching Strategies** (Gitflow, Trunk-based development).
- [ ] Handle **Merge Conflicts** and complex **Rebases** with confidence.
- [ ] Explore **Hooks** and **GitHub Actions** for CI/CD.

## 🛠 Tech Stack
- **VCS:** Git
- **Hosting:** GitHub / GitLab
- **Tools:** Oh My Zsh (Aliasing), GitKraken (Visualization), VS Code

## 📁 Repository Structure
- `/basics`: Essential commands and configuration.
- `/workflows`: Practice logs for branching and PR (Pull Request) reviews.
- `/scenarios`: Documentation on fixing mistakes (Reflog, Amending).
